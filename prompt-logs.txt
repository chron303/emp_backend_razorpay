Authentication:
- Cookie-based JWT authentication.
- Login must set an auth cookie.
- Protected routes will later read the auth cookie.
- JWT should contain userId and role.

Role Rules:
- Every newly registered account starts with EMP role.
- Users cannot choose their role during registration.
- Only emails ending with @org.com are allowed.

Endpoints:

POST /rest/onboardings/register

Request Body:
{
  "name": "John Doe",
  "email": "john@org.com",
  "password": "password123"
}

Rules:
- Validate required fields.
- Validate org.com domain.
- Email must be unique.
- Hash password before storing.
- Assign EMP role automatically.
- Do NOT log the user in.

POST /rest/onboardings/login

Request Body:
{
  "email": "john@org.com",
  "password": "password123"
}

Rules:
- Validate required fields.
- Validate org.com domain.
- Verify credentials.
- Generate JWT.
- Set auth cookie.
- Return success response.

POST /rest/onboardings/logout

Rules:
- Clear auth cookie.
- Return success response.
